Form Actions

The idiomatic SvelteKit way to handle form submissions and server-side mutations — no extra API endpoints required.

⏱ ~25 min 🎯 Core concept 📋 Requires: routing + load functions (done)

Why form actions exist

In your current project, mutations (login, register) follow this pattern: a <form> or button calls a fetch() to a +server.ts endpoint, which handles the logic and returns JSON.

Your current approach

// client-side JS required async function handleLogin() { const res = await fetch('/api/login', { method: 'POST', body: JSON.stringify(formData) }); // handle response... }

With form actions

// No JS needed for basic case // +page.server.ts handles it <form method="POST"> <input name="email" /> <button>Submit</button> </form>

Form actions give you three things the fetch-API approach doesn't:

  • Progressive enhancement — works without JavaScript enabled (the browser just submits the form natively)
  • Co-location — the server logic lives right next to the page that uses it, in the same +page.server.ts
  • Built-in data flow — validation errors and return values flow back to the component automatically via form prop
🔄
Coming from Nuxt? There's no direct Nuxt equivalent. The closest analogy is a server function + useFetch, but form actions handle the full request-response cycle including validation errors and redirects, all without explicit fetch calls in your component.

Anatomy of a form action

A form action is an exported actions object in +page.server.ts. Each key in the object is a named action (or default for a single action).

Server side: +page.server.ts

src/routes/notes/+page.server.ts import { fail, redirect } from '@sveltejs/kit'; import prisma from '$lib/server/db'; import type { Actions } from './$types'; export const actions: Actions = { // The 'default' action handles a plain POST to this page default: async ({ request, cookies }) => { const data = await request.formData(); const body = String(data.get('body') ?? '').trim(); // Validation: return errors without throwing if (!body) { return fail(400, { error: 'Note body is required.', body }); } await prisma.note.create({ data: { body } }); // Returning nothing re-runs the load function automatically } };

Client side: +page.svelte

src/routes/notes/+page.svelte <script lang="ts"> // 'form' is the data returned from the action (errors, etc.) let { data, form } = $props(); </script> <!-- method="POST" is all that's needed to wire the form --> <form method="POST"> {#if form?.error} <p class="error">{form.error}</p> {/if} <textarea name="body" value={form?.body ?? ''}></textarea> <button>Add note</button> </form> {#each data.notes as note} <p>{note.body}</p> {/each}
💡
Key: form vs data data comes from the load function and holds your initial page data. form comes from the action and holds whatever the action returned — typically validation errors or success state. Both are passed as props by SvelteKit automatically.

Named actions

A single page can have multiple actions. Use the ?/actionName query parameter syntax in the form's action attribute:

src/routes/notes/+page.server.ts export const actions: Actions = { create: async ({ request }) => { // handles POST ?/create }, delete: async ({ request }) => { const data = await request.formData(); const id = String(data.get('id')); await prisma.note.delete({ where: { id } }); } };
src/routes/notes/+page.svelte <!-- Targets the 'create' action --> <form method="POST" action="?/create"> ... </form> <!-- Targets the 'delete' action --> {#each data.notes as note} <form method="POST" action="?/delete"> <input type="hidden" name="id" value={note.id} /> <button>Delete</button> </form> {/each}
✅
Automatic data refresh When an action succeeds (returns nothing or a plain object), SvelteKit automatically re-runs the page's load function — so your list of notes updates without any manual state management. This is the biggest quality-of-life win.

Progressive enhancement with use:enhance

Form actions work without JavaScript. But with JavaScript, you can opt in to a smoother experience using the enhance action from $app/forms:

src/routes/notes/+page.svelte <script lang="ts"> import { enhance } from '$app/forms'; let { data, form } = $props(); </script> <!-- use:enhance intercepts the submit and sends it via fetch --> <!-- No page reload. Errors and data still flow the same way. --> <form method="POST" use:enhance> ... </form>

With use:enhance you also get a callback to handle loading state:

let submitting = $state(false); <form method="POST" use:enhance={() => { submitting = true; return async ({ update }) => { await update(); // re-run load + apply form result submitting = false; }; }} > <button disabled={submitting}> {submitting ? 'Saving…' : 'Save'} </button> </form>
⚠️
Don't forget await update() If you provide a callback to use:enhance, you must call update() yourself — SvelteKit won't do it automatically anymore. Without it, the load function won't re-run and your UI won't refresh.

Returning errors: fail() vs throwing

There are two ways to signal a problem from an action:

Throw — for unexpected errors

import { error } from '@sveltejs/kit'; // Shows the error page throw error(500, 'DB is down');

fail() — for validation errors

import { fail } from '@sveltejs/kit'; // Returns data to the 'form' prop return fail(422, { error: 'Email already taken' });

Use fail() for user-facing validation errors. The component receives the data in its form prop and can display it inline. Use throw error() only for genuine server errors that should show an error page.

Hands-on exercise

Build a simple note-taking page using what you just learned. ~15 minutes.

1
Add a Note model to Prisma

In prisma/schema.prisma, add a Note model with fields: id (cuid), body (String), createdAt (DateTime). Then run npx prisma migrate dev --name add-note.

2
Create src/routes/notes/+page.server.ts

Export a load function that fetches all notes from Prisma. Export an actions object with a default action that reads body from formData, validates it's not empty using fail(), and creates a new note.

3
Create src/routes/notes/+page.svelte

Destructure data and form from $props(). Add a <form method="POST"> with a textarea named body and a submit button. Display form?.error if it exists. List data.notes below the form.

4
Add use:enhance

Import enhance from $app/forms and add use:enhance to the form. Optionally add a submitting state variable to disable the button while saving.

5
Test it

Navigate to http://localhost:5173/notes. Submit an empty form — you should see your validation error. Submit a real note — it should appear in the list without a page reload.
