# Mission: SvelteKit

## Why
Add SvelteKit to a professional toolkit for building full-stack web apps — data-driven applications with databases, auth, and server-side logic. The goal is to be fluent enough to use SvelteKit as a complete stack on client work or personal projects, not just to understand it abstractly.

## Success looks like
- Build and ship a complete full-stack SvelteKit app (auth, database, server logic, deployment)
- Write idiomatic SvelteKit — form actions, hooks, load functions — not a "ported Vue app"
- Understand how SvelteKit differs from Nuxt and when/why to choose it
- Debug server/client data flow issues confidently
- Set up auth, protected routes, and middleware in a maintainable way

## Constraints
- Coming from Vue/Nuxt — analogies to that mental model are useful
- Existing project (`learn-sveltekit`) is the practice workspace
- Svelte 5 runes syntax is already in use in the project

## Out of scope
- Svelte 4 / legacy options API syntax (project is already on Svelte 5)
- Deep Vite/build tooling internals
- CSS frameworks (project uses plain SCSS)
